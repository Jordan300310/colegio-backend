# Cambios realizados (27-05-2026)

## Nuevos endpoints

### Secciones (asignaciones masivas)
- POST /secciones/{idSeccion}/alumnos/lote
  - Inscribir varios alumnos en una seccion.
  - Body: { "idsAlumno": [1,2,3] }
- DELETE /secciones/{idSeccion}/alumnos/lote
  - Dar de baja varios alumnos de una seccion.
  - Body: { "idsAlumno": [1,2,3] }
- POST /secciones/profesor/{idProfesor}/secciones
  - Asignar un profesor a varias secciones.
  - Body: { "idsSeccion": [1,2,3] }
- DELETE /secciones/profesor/{idProfesor}/secciones
  - Remover un profesor de varias secciones.
  - Body: { "idsSeccion": [1,2,3] }
- GET /secciones/{idSeccion}/profesor/{idProfesor}/alumnos
  - Lista alumnos de una seccion validando que el profesor este asignado.

### Progreso
- GET /progreso/no-avanzan
  - Query params: idSeccion, dias (default 7), minAvance (default 50)
  - Criterio: avance < minAvance y ultima actividad <= hoy - dias.

### Reportes (query params)
- GET /reportes/progreso
  - Query params:
    - idSeccion (obligatorio)
    - idAlumno (opcional)
    - formato = pdf | excel
  - Si no hay idAlumno: reporte grupal por seccion.
  - Si hay idAlumno: reporte individual del alumno dentro de la seccion.

### Cursos del alumno para profesor/admin
- GET /cursos/alumno/{idAlumno}
  - Profesor: solo ve cursos donde esta asignado.
  - Admin: ve todos los cursos del alumno.

## Cambios de logica
- Operaciones masivas: si algun ID falla, se cancela todo y se devuelve error con IDs fallidos.
- Reporte individual por seccion: se obtiene el curso desde la seccion y se valida que el alumno este matriculado.
